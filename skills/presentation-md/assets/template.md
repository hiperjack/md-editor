# md-editor プレゼンモード

Markdown を書くだけで、そのまま16:9スライドとして発表できる

---

## このデッキについて

これは1枚の Markdown ファイルです。プレゼンモードで開くと、ここまでの内容が複数のスライドに分かれて表示されます。

- 普段どおり Markdown を書くだけ。専用の書式や変換は不要
- HTMLプレビューと同じ見た目が、そのままスライドになる
- このファイル自体が、書き方のお手本（テンプレート）です

---

## スライドの基本構造

各スライドは「見出し → 結論メッセージ → 本文」の3つで組み立てます。

- **見出し** … スライド上部にタイトルとして固定表示される
- **結論メッセージ** … 見出し直後の最初の1段落。アクセント下線付きで強調される
- **本文** … それ以降すべて（箇条書き・表・図など）

この段落のように、見出しのすぐ下に1文を置くと「そのスライドで言いたいこと」が目立ちます。

---

## スライドの区切り方

スライドは区切り線で分けます。区切り線の前には必ず空行を入れてください。

1. `---`（区切り線）を置いた場所で次のスライドに切り替わる
2. 区切り線を書かなくても、見出し（H1・H2）の位置で自動的に切り替わる
3. スライドの途中で小見出しが欲しいときは `###` 以下を使う

> [!TIP]
> 「スライドの境目は区切り線、スライド内の小見出しは ### 」と決めておくと迷いません。

---

## 表も使えます

表はそのまま整形され、列が多くて収まらない場合は横スクロールになります。

| 操作 | キー | 説明 |
|---|---|---|
| 次のスライド | → / Space | 1枚進む |
| 前のスライド | ← | 1枚戻る |
| 一覧表示 | G | 全スライドをタイル表示 |
| フルスクリーン発表 | F / F5 | 画面いっぱいに表示 |

要点を絞った小さめの表ほど、スライドでは読みやすくなります。

---

## 図（Mermaid）も描けます

コードブロックに `mermaid` と書くと、フロー図やシーケンス図として描画されます。

```mermaid
flowchart LR
  A[Markdownを書く] --> B[プレビューで確認]
  B --> C[プレゼンモードに切替]
  C --> D[そのまま発表]
```

文章だけでは伝わりにくい流れや関係は、図にすると一目で伝わります。

---

## コードもそのまま見せられます

言語を指定したコードブロックは、シンタックスハイライト付きで表示されます。

```javascript
function greet(name) {
  // 1スライドに載せるコードは要点だけに絞る
  return `Hello, ${name}!`;
}
```

長いコードは縮小されて読みにくくなるため、要点の数行だけを載せるのがコツです。

---

## 詰め込みすぎないのがコツ

1スライドは1メッセージに絞ると、伝わるスライドになります。

- 箇条書きは1枚あたり5〜7行を目安にする
- 入りきらない内容は、思い切って次のスライドへ分ける
- 本文が多すぎると自動で縮小され、限界を超えるとスライド内スクロールになる

スライドは「読む資料」ではなく「見せる資料」。情報量より、伝わりやすさを優先します。

---

## まとめ

Markdown を書く力が、そのまま発表資料を作る力になります。

- [x] 見出し＋結論メッセージ＋本文の3ゾーンで書く
- [x] スライドは空行付きの `---` で区切る
- [x] 表・図・コード・チェックリストもそのまま使える
- [ ] あとは、あなたの内容で1枚作ってみる

このファイルを土台に、中身を書き換えるところから始めてみてください。
